What Is This?

Imagine the iconic Matrix “digital rain” — but instead of Japanese katakana, your screen fills with IMMUTABILITY, RANSOMWARE RECOVERY, ZERO-TRUST RESILIENCE, and VEEAM DATA CLOUD VAULT cascading down in glowing green. That’s VeeaMatrix: a fully functional Windows screensaver (.scr) built entirely in C#, compiled without Visual Studio, and packed with more Veeam buzzwords than a Keynote slide deck.

It started as a side project. It became an obsession. Twenty-six versions later, it has a live preview, a multi-tab settings dialog, Light/Dark mode, five built-in color profiles, embedded banner support, and a 60+ term catalog covering everything from VBR V13 to AI TRiSM.

Technical Deep-Dive

For the curious. Skip if you just want the screensaver.

Single-file, no IDE

The entire screensaver is one VeeaMatrix.cs file (~2,100 lines), compiled via the .NET Framework 4.x command-line compiler. The build script (Build-VeeaMatrix.ps1) optionally embeds a banner image as a compiled resource — so the final .scr is truly self-contained.

csc.exe /target:winexe /out:VeeaMatrix.scr /r:System.Windows.Forms.dll,System.Drawing.dll /optimize+ VeeaMatrix.cs

Rendering loop

Each frame: fill with a semi-transparent black overlay → draw each active column glyph → overlay active word popups → draw watermark → BufferedGraphics.Render() to screen. Font sizes scale from screen height for clean rendering at 1080p, 1440p, and 4K.

Popup state machine

Each popup cycles: Idle → Appearing → Visible → Fading → Done. Five modes differ in rendering — Glitch shuffles characters, Scan draws a horizontal reveal bar, Zoom scales from 0 to full, Scramble resolves noise left-to-right.

Settings architecture

The ConfigForm uses a RebuildUI() → Build() pattern: switching language or toggling Light/Dark mode recreates all controls from scratch. Thirteen theme color fields are computed once from cur.DarkMode at the top of Build().

How It Evolved

Phase 1 · v1.0–v1.10

"Does this even work?"

Can you compile a working screensaver from a single C# file without Visual Studio? The answer is yes — csc.exe ships with .NET Framework. Early versions proved the rendering loop and wired up the three standard entry points (/s, /p, /c).

Phase 2 · v1.11–v1.18

Bringing it to life

Word streams, five popup animation modes, multi-monitor support, font controls, and the first color profiles.

Phase 3 · v1.19–v1.22

Settings UI & polish

Three-column settings dialog with live 16:9 preview, Light/Dark mode toggle, and banner image compiled directly into the .scr.

Phase 4 · v1.23–v1.26

Final refinements

Term catalog expanded with VBR V13, VDP tiers, Veeam Recovery Orchestrator, RECON SCANNER, AGENT COMMANDER, AI TRiSM, and more. Bold/Italic font flags, Deep Space profile, Veeam 100 Easter-egg mode, and content isolation (Veeam terms only show when at least one Veeam source is enabled).

Built with AI

This project was developed entirely through a conversation with Claude (Anthropic). No IDE, no StackOverflow, no manual debugging — just a chat window, a text editor, and Build-VeeaMatrix.ps1.

  • Iterative refinement — describe a visual effect in natural language, get working GDI+ code back
  • Architecture decisions — RebuildUI() pattern, FontStyle flags, embedded-resource strategy all emerged from discussion rather than upfront design
  • Staying in .NET 4.x — no LINQ, no NuGet. The constraints kept the code clean and the build trivial

Installation

Quick start:

  1. Download VeeaMatrix.scr
  2. Double-click → screensaver runs immediately
  3. Right-click → Configure for the settings dialog

System-wide: Right-click VeeaMatrix.scr → Install (or copy to C:\Windows\System32\ as Admin)

Build from source:

.\Build-VeeaMatrix.ps1   # place VeeaMatrix-banner.jpg next to it to embed your photo

Requires .NET Framework 4.x (ships with Windows 10/11).
